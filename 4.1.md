## Контекст задачи

Ты работаешь с backend проекта Kanban на Django + Django Ninja.

Нужно выполнить backend-часть задачи 1 Sprint/Этап 4:

## Основная идея реализации

REST endpoint’ы остаются основным способом изменить данные.

WebSocket используется для доставки realtime-событий после успешных изменений.

Схема:

```text
frontend вызывает REST endpoint
backend проверяет права
backend меняет БД
backend возвращает REST response
backend отправляет WebSocket-событие пользователям открытой доски
```

Это нужно, чтобы не дублировать бизнес-логику в WebSocket consumer и не превращать WebSocket в отдельный CRUD API.

## WebSocket endpoint

Добавить один основной WebSocket endpoint доски:

```text
/ws/boards/{board_id}/
```

Подключение должно быть доступно только пользователям, у которых есть доступ к этой доске.

Авторизация через JWT:

```text
/ws/boards/{board_id}/?token=<access_token>
```

При подключении нужно:

```text
получить board_id
проверить JWT
получить пользователя
проверить доступ пользователя к board
подключить пользователя к группе этой доски
```

Если пользователь не авторизован или не имеет доступа к доске, соединение закрывается.

## Формат WebSocket-сообщения

Использовать единый формат события.

Пример структуры:

```json
{
  "type": "task.moved",
  "board_id": "uuid",
  "payload": {},
  "actor_id": "uuid-or-null",
  "created_at": "iso-datetime"
}
```

Требования к формату:

```text
type — строковый тип события
board_id — доска, в рамках которой произошло событие
payload — данные, необходимые frontend для точечного обновления
actor_id — пользователь, который вызвал изменение, если его можно определить
created_at — время события
```

Payload должен быть достаточно полным для точечного обновления, но без лишней вложенности и без передачи всей доски целиком.
